GetSmartLook analýza¶

Tento dokument ukazuje analzu dt od GetSmartLook.

Zdroj a rozsah dát¶

Od GetSmartLook sme dostali cez 10GB dt v JSON formte, ktor reprezentuj jednotliv "sessions". Dokopy mme 88634 "sessions".

Tieto dta s z obdobia medzi 6. a 12. mjom 2016 a pochdzaj predovetkm zo strnok GetSmartLook.

Spracované, agregované dáta¶

Surov JSON dta sme spracovali parserom, ktor sme napsali na zklade ukkovho JSON sboru, ktor sme dostali vopred. Z tohto parseru sme dostali csv sbory, v ktorom s uloen vetky "udalosti" ako kliky, pohyby, scrolls.

Nsledne sme tieto dta spracovali do agregovanch tatistk na rovni jednotlivch sessions. To znamen, e na zklade udalost z vyparsovanch csv sborov sme pre kad session vypotali premenn ako "vzdialenos", "frekvencia klikov", at.

Tento dokument uke, o sa momentlne s dtami d robi, ako sme ich spracovali a ak prv analzy a poznatky z nich vieme zohna. Zatia na menej vzorke.

Výpočet štandardov (baselines)¶

Ke sme natali a pospjali agregovan dta, meme na zklade tchto sessions vypota tandardy sprvania. Spravme tak globlne, ako aj zvl pre kad server, ktor v dtach mme.

tandardy (baselines) slia na identifikciu "najzvltnejch" (a teda najzaujmavejch) sessions.

Kee pre kad premenn vypotame priemer aj tandardn odchlku, dokeme pre kad session vypota, ako aleko s od priemeru (koko tandardnch odchliek). Ak bude tto hodnota pre akkovek premenn vrazne odlin (sigma aspo 4, teda 4 tandardn odchlku), ozname si ich. Tak dokeme identifikova sessions, ktor maj pri najvom pote premennch vrazn odchlky.

Takto postup vieme aplikova aj na rovni serveru - pre kad server vieme vypota odlin tandardy a porovnva tak sessions poda vhodnch tandardov.

import numpy as np

def t_test_baselines(server1, server2, sigma=5):
    """
    T-test rozdielov medzi standardami dvoch serverov
    Sigma urcuje pozadovanu signifikanciu rozdielu
    """
    # standardy oboch serverov
    baseline1 = server_baseline_df.ix[server1]
    baseline2 = server_baseline_df.ix[server2]
    # pocet sessions pre kazdy server
    sess_count1 = server_counts[server1]
    sess_count2 = server_counts[server2]
    # priemery
    means1 = baseline1.ix['mean'].values
    means2 = baseline2.ix['mean'].values
    # variance
    variance1 = baseline1.ix['variance'].values
    variance2 = baseline2.ix['variance'].values
    # standardna chyba
    se = np.sqrt( (variance1 / sess_count1) +  (variance2 / sess_count2) )
    # rozdiel priemerov
    mean_diff = means1 - means2
    # t hodnoty
    test_result = mean_diff / se
    # kolko je viac ako sigma
    t_abs = np.abs(test_result)
    sig = t_abs > sigma
    return test_result, 1. * np.sum(sig) / len(sig)

    first = np.dot(inv_sigma, x_minus_mu.T)
